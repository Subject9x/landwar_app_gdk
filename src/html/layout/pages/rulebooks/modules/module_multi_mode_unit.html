Multi-Mode / Alt-Mode Units

Multi-mode rule allows players to take several different Unit cards and combine them into different modes. An Example unit would be a mech that could transform into a flying type of Unit.

Begin by building each mode as a standard unit.

  • Each mode is a separate Unit entry, and tags are not shared by default (see exceptions).
  • Unit's total cost is the cost of the highest mode's points cost including tags.
  • Unit modes may have different Armor values, but see below for caveats.

  • Initiative Phase
    • Before all players roll initiative, Players must declare if Unit is switching from one mode to another.
    • IF Unit transforms, Unit cannot use any initiative tags this phase.
    • Panicked Units cannot transform.
    • IF a Unit tranforms to a mode with less total Armor than it currently has, unit is destroyed.
  • Damage
    • Unit takes damage as normal, when total damage exceeds any given mode, the Unit cannot switch to that mode without being destroyed.
  • Stress
    • Stress is tracked once for all modes.
    • Only the active Unit mode's tags may affect the Unit's Stress value and checks.
  • Tag Interactions

    if tag is not listed here, assume the tag is applied as normal.

    • [Area Denial]: Place once on any 1 mode, applies to all modes.
    • The following can only applied to the highest costed mode.
      • [Courage-I], [Courage-II], [Crew-I], [Crew-II]
      • [Hero], [Mobile HQ], [Rank-Green], [Rank-Veteran], [Rank-Elite]
